# LinkedIn Automation & Chat Tracker

This repository contains scaffolding and integration scripts designed to generate and build a full-stack Next.js application for a **LinkedIn Chat Tracker**. It automates setting up a user interface, database scheme, and an extensive Unipile API integration for LinkedIn connectivity.

## Files In This Directory

### **`scaffold.js`**
A Node.js script that rapidly generates the foundational structure and necessary boilerplate for the `linkedin-chat-tracker` Next.js application. 
- Creates key configuration files (`package.json`, `tsconfig.json`, `tailwind.config.ts`, `next.config.ts`).
- Sets up a robust **Prisma Schema** with models for `User`, `LinkedInAccount`, `Contact`, `Conversation`, `Message`, `ActivityLog`, and `MessageTemplate`.
- Generates empty frontend component and API route stubs, preparing the project for phase 2.
- Handles setting up essential UI libraries (like Radix UI, Tailwind CSS, Zustand and React Query).

### **`scaffold2.js`**
An alternative version of the scaffolding script. It achieves the exact same goals but formatting and stringification methods differ slightly (using `JSON.stringify` instead of pure template literals in some places). Run this or `scaffold.js` from the root directory to instantiate the base project before proceeding to Phase 2.

### **`phase2.js`**
The core script providing the API integration logic for the project. By running this script, you overwrite the stubs generated by the initial scaffolding with fully functional code tailored for **Unipile**. It generates:
- **`src/types/unipile.ts`**: Dedicated TypeScript definitions for Unipile models (`UnipileAccount`, `UnipileMessage`, etc.).
- **`src/lib/unipile.ts`**: The `UnipileClient` wrapper handling polling, rate limits, paginated queries, searches, and error reporting.
- **API Routes (`src/app/api/...`)**: Working REST endpoints mapping the Unipile client data layer with Prisma. This includes connecting LinkedIn accounts, fetching chats and messages, sending messages to connections or profiles, sending connection requests, and handling webhook events (`new_message`, `connection_accepted`).

### **`linkedin-automation/` (Directory)**
A subdirectory designed to maintain specific automated scripts or the target environment. You will likely find the resultant generated `linkedin-chat-tracker` project folder inside here depending on your execution directory.

## How to use

1. Run \`node scaffold.js\` to initialize the Next.js `linkedin-chat-tracker` base.
2. Navigate into the generated directory folder mappings.
3. Run \`node phase2.js\` (adjust paths if required) to inject the Unipile database schemas and Next.js backend logic functionality over the stubs. 
4. Don't forget to configure the \`.env\` or \`.env.local\` keys generated (such as `UNIPILE_DSN`, `UNIPILE_ACCESS_TOKEN`, `DATABASE_URL`).
5. Run the standard Next.js sequence: `npm install`, `npx prisma db push`, `npm run dev`.
